# Clip Bridge 设计文档

## 概述

跨平台剪贴板共享工具，用于在局域网内多台电脑之间同步纯文本剪贴板内容。

### 设计目标

- Mac 和 Ubuntu 之间双向同步纯文本剪贴板
- TCP Socket 双向通信，手动配置对端 IP
- 首次运行交互式配置，之后直接使用
- 防止循环同步

## 架构

### 整体架构

采用**单进程多线程**架构：

```
┌─────────────────────────────────────────────────────────────┐
│                     Clip Bridge 进程                         │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐    ┌──────────────────────────────────┐    │
│  │ Config      │───▶│        Main Thread               │    │
│  │ Loader      │    │  - 启动/管理所有线程              │    │
│  └─────────────┘    │  - 处理优雅退出                   │    │
│  ┌─────────────┐    │  - 防循环状态管理                  │    │
│  │ Interactive │    └──────────────────────────────────┘    │
│  │ Setup       │                                              │
│  └─────────────┘         ▲          ▲          ▲            │
│                          │          │          │            │
│  ┌─────────────┐        │          │          │            │
│  │ Event Queue │────────┘          │          │            │
│  └─────────────┘                   │          │            │
│                                     │          │            │
│        ┌─────────────┐  ┌──────────┴──┐  ┌───┴───────────┐ │
│        │  Monitor    │  │   Sender    │  │   Receiver    │ │
│        │  Thread     │  │   Thread    │  │   Thread      │ │
│        └─────────────┘  └─────────────┘  └───────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 选择理由

- 简单：多线程足够，不需要复杂的进程通信
- 高效：状态共享直接，防循环容易实现
- 可靠：线程间通信简单，错误处理集中

## 组件

### 1. config.py - 配置管理

```yaml
# 配置结构
local_port: 9999           # 本地监听端口
remote_host: 192.168.1.100 # 对端 IP
remote_port: 9998          # 对端端口
poll_interval: 0.5         # 剪贴板轮询间隔（秒）
sync_cooldown: 2.0         # 防循环冷却时间（秒）
max_size: 1048576          # 最大消息大小 1MB
```

### 2. monitor.py - 剪贴板监听器

- 使用 pyperclip 定期轮询（跨平台兼容性）
- 检测到变化后检查防循环状态
- 通过线程安全队列发送事件

### 3. sender.py - TCP 发送器

- 线程安全的发送队列
- 自动重连机制（指数退避）
- 消息格式：`CLIP<length>:<content>`

### 4. receiver.py - TCP 接收器

- 单一连接模式（拒绝额外连接）
- 解析消息格式
- 更新剪贴板并设置防循环状态

### 5. main.py - 入口

- 交互式配置向导（首次运行）
- 启动所有线程
- 信号处理（优雅退出）

## 协议

### 消息格式

```
CLIP<length>:<content>
```

- `CLIP` - 4字节前缀
- `<length>` - 内容长度（十进制）
- `:` - 分隔符
- `<content>` - 实际内容

### 最大消息大小

1MB

## 防循环机制

```python
# 使用时间戳 + 内容哈希去重
last_received = {}  # {content_hash: timestamp}

# 发送前检查
if is_cooldown_active():
    return  # 跳过发送

# 接收后设置冷却
record_received(content, current_time)
```

冷却时间内收到相同内容时，不再触发同步。

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 网络断开 | Sender 自动重连，Receiver 等待新连接 |
| 剪贴板异常 | 记录日志，继续监听 |
| 配置错误 | 启动时验证，失败则退出 |
| 消息格式错误 | 记录日志，关闭连接 |
